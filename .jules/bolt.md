## 2024-06-25 - O(N^2) Array Uniqueness Check using Set(UInt64)

**Learning:** `Array#includes?` inside loops (like `compact_types` iterating to add unique types to an array) results in an O(N^2) time complexity. Since Type objects in the Crystal compiler are reference types tracked by `object_id`, allocating a parallel `Set(UInt64)` eliminates the hashing overhead of using `Set(Type)` directly and makes tracking O(1) inside hot paths. However, allocating `Set` carries its own overhead, meaning small collections (e.g. `size < 15`) perform worse with the optimization.
**Action:** Use conditionally allocated Sets (e.g., `if size > 15`) tracking `object_id` when appending to an array of objects while ensuring uniqueness. Retain previous method signatures (via overloads passing the `Set` downwards) to avoid regressions.
