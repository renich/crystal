## 2024-05-24 - Optimizing type_merge array insertions using parallel Sets
**Learning:** In the Crystal compiler (`Program#compact_types`), deduplicating large collections of types using `Array#includes?` (`array << item unless array.includes?(item)`) creates an $O(N^2)$ bottleneck. Using a parallel `Set(UInt64)` to track `type.object_id` alongside the array insertion turns this into an $O(N)$ operation. We use `object_id` to avoid object hashing overhead and optimize performance since `Type` objects are interned.
**Action:** When maintaining collections of unique objects in performance-critical paths, retain the original method signatures via method overloading, but internally branch to a `Set`-assisted approach if the collection size exceeds a small threshold (e.g. 15) to offset the allocation overhead for small arrays.
