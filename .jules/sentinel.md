## 2024-07-14 - Windows Path Traversal via Backslashes
**Vulnerability:** The HTTP StaticFileHandler did not correctly block backslashes `\` from incoming request paths when running on Windows. Because Windows treats `\` as a directory separator, malicious inputs containing `\` could bypass standard path normalization protections, leading to path traversal.
**Learning:** URL paths should not contain backslashes. Relying solely on `Path` library normalization isn't enough if the framework accepts both `/` and `\` as separators on Windows. We must explicitly reject backslashes.
**Prevention:** Always validate and reject forbidden characters early in the request lifecycle. Use `{% if flag?(:win32) %}` to conditionally apply Windows-specific character checks to avoid breaking POSIX compatibility.
